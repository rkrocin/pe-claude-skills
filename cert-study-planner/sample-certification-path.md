# Certification Path: Platform Engineering Leader in Financial Services

## Profile

An infrastructure and platform engineering director targeting Director-level roles in Chicago financial services. Current stack spans hybrid cloud, IDP, DevSecOps, and AI enablement. Goal is to build a credential portfolio that signals depth across cloud architecture, security governance, AI platforms, and cloud financial management.

## Recommended Certification Sequence

### Tier 1: Foundation Layer (Already Strong)

These establish credibility across the core domains. If already held, they serve as the base for advanced certifications.

| Order | Certification | Rationale | Estimated Prep |
|-------|--------------|-----------|----------------|
| 1 | AWS Solutions Architect - Professional | Validates deep AWS architecture expertise at enterprise scale | 6-8 weeks |
| 2 | AWS Developer - Associate | Validates hands-on development capability alongside architecture | 3-4 weeks |
| 3 | CISSP | Gold standard for security leadership credibility in regulated industries | 8-10 weeks |

### Tier 2: Specialization Layer

These differentiate the profile by demonstrating depth in specific domains that matter for FSI platform roles.

| Order | Certification | Rationale | Estimated Prep |
|-------|--------------|-----------|----------------|
| 4 | ISSAP | Architecture-level security credential. Rare and highly valued in regulated environments. Requires CISSP. | 6-8 weeks |
| 5 | ISSEP | Engineering-level security credential. Complements ISSAP with implementation depth. | 6-8 weeks |
| 6 | CCSP | Cloud-specific security. Bridges the CISSP and AWS credential sets. Strong overlap with CISSP reduces prep time. | 4-5 weeks |
| 7 | CISM | Security management perspective. Signals readiness for security governance leadership. | 5-6 weeks |
| 8 | CRISC | Risk management focus. Directly relevant to financial services regulatory environment. | 5-6 weeks |
| 9 | CGEIT | Enterprise IT governance. Less common, signals board-level governance awareness. | 5-6 weeks |

## Sequencing Strategy

### Knowledge Cascade

The sequence above is designed so each certification builds on the one before it:

```
AWS SA Professional
    └── AWS Developer (overlapping AWS fundamentals)
        └── CISSP (adds security governance layer)
            ├── ISSAP (architecture-level security, builds on CISSP)
            ├── ISSEP (engineering-level security, builds on CISSP)
            └── CCSP (cloud security, merges CISSP + AWS knowledge)
                └── CISM (management layer, builds on CISSP concepts)
                    └── CRISC (risk layer, builds on CISM governance)
                        └── CGEIT (enterprise governance, builds on all above)

AWS GenAI Developer Professional
    └── AWS ML Specialty (deeper ML, builds on GenAI concepts)
        └── AWS ML Engineer (MLOps, builds on ML Specialty)
            └── AAISM (AI security, merges security + AI knowledge)

FinOps Practitioner
    └── FinOps Engineer (builds directly on Practitioner)
```

### Overlap Exploitation

Several certification pairs share significant content, reducing total study time when taken sequentially:

| Pair | Overlap | Time Saved |
|------|---------|------------|
| CISSP then CCSP | ~40% content overlap (security fundamentals, governance, risk) | 3-4 weeks |
| CISSP then CISM | ~30% overlap (governance, risk management, incident response) | 2-3 weeks |
| CISM then CRISC | ~25% overlap (risk frameworks, governance) | 2 weeks |
| AWS ML Specialty then GenAI Developer | ~20% overlap (SageMaker, model concepts) | 1-2 weeks |
| FinOps Practitioner then Engineer | ~30% overlap (principles, personas, phases) | 1-2 weeks |

### Pacing

- Space exams 4-6 weeks apart to allow focused preparation without burnout
- Group related certs (e.g., CISSP then CCSP back-to-back while the material is fresh)
- Plan for 1-2 certifications per quarter as a sustainable pace alongside full-time work
- Allow one "break quarter" per year with no exams

## Market Signal Analysis

For Director-level platform engineering roles in Chicago financial services:

**High signal certs** (frequently listed in JDs or valued by hiring managers):
- CISSP: near-universal in FSI security and infrastructure leadership
- AWS Solutions Architect Professional: validates cloud architecture depth
- CISM/CRISC: valued by ISACA-aligned organizations (banks, insurers, exchanges)

**Differentiating certs** (uncommon combinations that stand out):
- ISSAP + ISSEP: very few candidates hold both. Signals serious security architecture depth.
- AWS GenAI Developer + AAISM: AI capability with AI security governance. Rare and timely.
- FinOps Engineer: demonstrates budget management discipline with formal methodology.

**Table-stakes certs** (expected but not differentiating):
- AWS Solutions Architect Associate: assumed for any cloud role
- Security+: entry-level, only matters if no higher security cert is held
- Terraform Associate: helpful but not a differentiator at Director level
